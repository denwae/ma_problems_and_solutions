# ByteBuddy Plugin - `ClassNotFoundException`

## Meta
Status: Analysiert, Dokumentation angepasst
Ticket: https://github.com/xmolecules/jmolecules-integrations/issues/334

## Zusammenfassung

Beim ausf√ºhren des `transform` Tasks mit dem `jmolecules-bytebudd-nodep` Plugin, wird eine `ClassNotFoundException: org.hibernate.annotations.EmbeddableInstantiator` geworfen.

## Ursache
Falsche Konfiguration der ByteBuddy Plugin?

## L√∂sung
Anstelle, dass wie in viele Beispielen der ByteBuddy Plugin das jMolecules ByteBuddy Plugin, als Dependency, hinzugef√ºgt wird. Soll die Dependency nicht hinzugef√ºgt werden und die `classPathDiscovery` Config auf `true` gesetzt werden, wie in den [offiziellen Beispielen](https://github.com/xmolecules/jmolecules-examples).

### Code
```xml
<build>
	<!-- ommited for legibility -->
    <plugins>
	    <!-- ommited for legibility -->
        <plugin>
			<groupId>net.bytebuddy</groupId>  
            <artifactId>byte-buddy-maven-plugin</artifactId>  
            <version>${bytebuddy.version}</version>  
            <executions>
				<execution>  
                    <goals>  
                        <goal>transform-extended</goal> <!-- Enable the source code transformation -->  
                    </goals>  
                </execution>  
            </executions>  
            <configuration>
				<classPathDiscovery>true</classPathDiscovery>  
                <initialization>  
                    <validated>false</validated>
				</initialization>
			</configuration>
		</plugin>  
    </plugins>  
</build>
```
